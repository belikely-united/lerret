# Security Policy

## Supported versions

Lerret is in pre-release. While the project is on `0.x.y`, only the latest release receives security updates.

| Version | Supported |
|---------|-----------|
| Latest `0.x.y` | Yes |
| Older `0.x.y` | No |

After `1.0.0`, this table will list the specific versions under support.

## Reporting a vulnerability

**Please do not open a public issue for security reports.** Public disclosure before a fix is available puts users at risk.

Use one of the private channels below:

1. **GitHub Security Advisories** — open a private advisory at `github.com/belikely-united/lerret/security/advisories/new`. This is the preferred channel.
2. **Email** — `oss@belikely.com`.

Include in your report:

- A description of the issue and the impact.
- Steps to reproduce, or a proof-of-concept.
- Affected versions.
- Any suggested mitigation, if you have one.

## Response timeline

- **Acknowledgment:** within 72 hours of receipt.
- **Initial assessment:** within 7 days.
- **Fix or mitigation plan:** communicated within 30 days for high-severity issues.

We will keep you informed throughout the process and credit you in the release notes (unless you ask to remain anonymous).

## Disclosure policy

We follow coordinated disclosure: a fix is prepared privately, then released alongside a public advisory. We aim to publish the advisory within 90 days of the original report, or sooner if a fix ships first.

## Out of scope

- Issues in third-party dependencies — please report those upstream and let us know so we can track the patch.
- Vulnerabilities that require physical access to a user's machine.
- Social-engineering attacks against contributors or users.

Thanks for helping keep Lerret and its users safe.
