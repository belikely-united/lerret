# CLI Reference

This page is a stub. Each command will get a full reference card.

## Running the CLI

The CLI is published as **`@lerret/cli`** on npm. The binary it installs is named `lerret`.

| Context | Command |
|---|---|
| Zero-install (`npx`) | `npx @lerret/cli dev` |
| Installed as a project dep | `lerret dev` (via `package.json` scripts, `pnpm exec`, `npx`, etc.) |
| Installed globally | `lerret dev` |

> Don't run `npx lerret …` — that bare name belongs to a different (unrelated, deprecated) npm package. Always include the `@lerret/cli` scope when invoking via `npx`.

The command syntax shown below uses the post-install form (`lerret …`). Prefix with `npx @lerret/cli ` if you haven't installed it.

## `lerret dev`

Runs the studio against a `.lerret/` project folder.

```bash
lerret dev [path-to-project] [--port <port>] [--no-open]
```

## `lerret export`

Headlessly renders a project (or page/group) to image files.

```bash
lerret export [path-to-project] --out ./out --format png
lerret export [path-to-project] --out ./out --flat
lerret export [path-to-project] --out ./out --page social
```

Full flag list lands in a follow-up commit.

## `create-lerret`

Scaffolds a new Lerret project.

```bash
npx create-lerret <project-name> [--no-samples]
```
