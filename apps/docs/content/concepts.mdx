# Concepts

This page is a stub. Topics that will land here:

- **The folder-canvas model** — folder = page, file = asset.
- **`.lerret/` structure** — the project marker; what's inside; what's ignored.
- **Pages and groups** — top-level folders are pages; nested folders are groups.
- **Reserved folders** — `_fonts/`, `_assets/`, and other `_`-prefixed names.
- **Assets and variants** — named exports become per-asset variants.
- **The `meta` export** — sizing, labeling, tagging, `propsSchema` for editor forms.
- **Data files** — `.data.json` / `.data.js` co-located with the asset; per-variant data keys.
- **The config cascade** — per-folder `config.json` files merged top-down; `vars` as CSS custom properties.
- **The four-tier prop resolution** — data → config → schema default → function default.
- **Live refresh** — file-watcher → per-asset HMR; sub-second re-render.
- **Error isolation** — one broken artboard fails alone, the rest of the canvas keeps rendering.

For now, peek at the sample project generated by `npx create-lerret` to see the patterns in practice.
