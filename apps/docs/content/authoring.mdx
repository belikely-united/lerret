# Authoring Assets

This page is a stub. Topics that will land here:

- **JSX components** — what a Lerret asset looks like, named exports as variants.
- **`meta` export** — sizing, tags, labels, `propsSchema` for the in-studio data editor.
- **Props vs. data** — the cascade, when to use which.
- **Markdown assets** — `.md` files as document cards.
- **Fonts** — drop `.woff2` / `.woff` into `_fonts/`; auto-registered.
- **Resources** — import images, JSON, CSS like in any Vite project.
- **The separation invariant** — Lerret code never touches your component files; your components never depend on Lerret.

For now, the scaffolded project's `.lerret/social/` directory has working examples of every authoring pattern.
